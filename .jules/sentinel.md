## 2024-05-24 - Hardcoded API Key in Frontend

**Vulnerability:** A hardcoded API key (`swarm_dev_key`) was present in the frontend JavaScript codebase (`src/api/static/app.js`). This allowed anyone with access to the static frontend files to obtain the key and make authenticated requests to the backend API.
**Learning:** Hardcoding secrets in client-side code is a common anti-pattern, especially during early development or MVPs. It often happens when developers prioritize getting a feature working quickly over implementing proper authentication flows. In this case, the `SWARM_API_KEY` was intended to be configured via an environment variable on the server, but the frontend simply used the default developer key. Furthermore, the API server handles missing configuration by returning a 500 error instead of failing securely, which caused confusing behavior for the frontend implementation.
**Prevention:** Never hardcode secrets in client-side code. Implement secure configuration management and authentication mechanisms. For simple setups, use prompt-based configuration stored securely (e.g., in `localStorage` for temporary development/MVP use cases, though more robust solutions are needed for production). Always validate configuration early and fail securely with clear error messages rather than generic 500 errors.
